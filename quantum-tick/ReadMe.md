# QuantumTick

This project follows standard Java and Maven conventions. Please adhere to the guidelines below when contributing.

## 📂 Project Structure (Where Things Go)

We follow the standard Maven directory layout. Do not put source code in the root directory.

* `src/main/java/com/group1` : **Main Source Code.** All your Java classes go here.
* `src/main/resources/com/group1` : **Non-code files.** Configuration files (e.g., `application.properties`), images, and static assets go here. They are bundled into the JAR automatically.
* `src/test/java/com/group1` : **Test Code.** All JUnit/TestNG testing files go here. They mirror the package structure of the main code.
* `src/test/resources/com/group1` : **Test assets.** Files needed only for running tests.
* `target/` : **Build output.** Auto-generated by Maven. Do not edit or commit this folder to Git.
* `pom.xml` : **Project Object Model.** The heart of Maven. All dependencies and build plugins are defined here.

## 📦 Package Conventions

Our base package is: `com.group1` (Reverse domain name format).

**We package by feature, not by layer.** If you are building a "User Registration" feature, keep the relevant classes together:
* ✅ **DO:** `com.yourname.myapp.user` (Contains `User.java`, `UserService.java`, `UserRepository.java`)
* ❌ **DON'T:** `com.yourname.myapp.models`, `com.yourname.myapp.services` (This scatters feature code across the app).

## ✍️ Coding Style & Naming Conventions

* **Classes & Interfaces:** `PascalCase` (e.g., `CustomerAccount`, `DatabaseConnector`). Nouns only.
* **Methods:** `camelCase` (e.g., `calculateTotal()`, `getUserName()`). Should be verbs/actions.
* **Variables:** `camelCase` (e.g., `firstName`, `orderCount`). Keep them descriptive. No single-letter variables except in short loops (e.g., `for (int i=0;...)`).
* **Constants:** `UPPER_SNAKE_CASE` with the `static final` modifiers (e.g., `public static final int MAX_RETRIES = 3;`).
* **Braces:** K&R style (opening brace on the same line as the declaration).
    ```java
    public void myMethod() {
        if (condition) {
            // do something
        }
    }
    ```

## 🚀 How to Build and Run

To compile the code, bundle dependencies, and generate an executable `.exe` and `.jar`:

1. Open a terminal in the project root.
2. Run: `mvn clean package`
3. Check the `/target` folder for the built executable.

## Maven not downloaded?
You do not need to install Maven on your machine to build this project! We use the Maven Wrapper, which will automatically download everything you need.

1. Clone this repository to your local machine.
2. Open a terminal in the root folder of the project.
2. Open a terminal in the root folder of the project.
3. Run the build command using the wrapper:
   * **Windows:** `.\mvnw.cmd clean package`
   * **Mac/Linux:** `./mvnw clean package`
4. The compiled executable and JAR will be generated in the `/target` directory.
